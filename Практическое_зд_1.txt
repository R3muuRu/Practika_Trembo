MVVM - это архитектурный паттерн, 
который используется при разработке ПО
для обеспечения разделения логики приложения, пользовательского 
интерфейса и бизнес-логики.

Схема взаимодействия серверной и клиентской частей мобильного приложения:

1. Клиентское приложение отправляет запрос на серверную часть через сетевой протокол.
2. Серверная часть получает запрос, обрабатывает его и возвращает результат в виде данных.
3. Клиентское приложение получает данные от сервера и передает их в ViewModel.
4. ViewModel обрабатывает полученные данные, выполняет необходимые операции и обновляет Model с помощью 
сетевых запросов.
5. Обновленные данные из Model передаются в View для отображения пользователю.
